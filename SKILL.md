## Document Generation Flow (CRITICAL — No External Signing Platform)

Documents are generated by the codebase at three CTA trigger points. There is no DocuSign or
external signing ceremony.

```
CTA 1: Consent Page  →  AUTH packet generated
CTA 2: Submit        →  APPLICATION packet generated
CTA 3: Payment       →  POLICY packet generated
```

- Signing is handled entirely by the codebase at each CTA
- Policy Number is assigned when the POLICY packet is generated — record in M&T log
- Auth packet **must be generated first** — verify it precedes application and policy packets

## Section 1 — Live PDF Document Testing

### Test Configuration (Required Before Every Run)
Before analyzing any PDF, confirm and record:
- [ ] **Product** — which of the 6 products
- [ ] **State** — which US state the policy is issued in
- [ ] **Form Control** — active control for that product (see Products Reference)
- [ ] **Channel** — DTC or AA
- [ ] **Replacement** — YES or NO
- [ ] **RUW Policy** — YES or NO (LGA Prime / Banner SIWL)
- [ ] **Scenario ID** — e.g. LGA-01, IUL-05, CHC-12, FEGI-03
- [ ] **Policy Number** — assigned at payment CTA; record in M&T log
- [ ] **Tester Name** — for M&T log attribution

### Packet Types

| Packet | Generated At | Forms Included |
|--------|-------------|----------------|
| Auth Packet | Consent page CTA | E-Sign · HIPAA · FCRA · NOIP · TXT · MIB · RTA · RPLC B |
| Application Packet | Submit CTA | Application · Carrier HIPAA · ADB/ABR Disc · Buyer's Guide · Replacement (conditional) |
| Policy Packet | Payment CTA | Contract · Riders · Perks · Delivery Receipt · State Endorsements |

### Critical Field-Check Rule (ALWAYS APPLY)
> ⚠️ **Field checks are RULES, not hardcoded expected values.**
> - NEVER assert "expected value = X"
> - ALWAYS verify each field matches the actual funnel response for that specific test applicant
> - Medical question responses must each be verified individually — mismatch is a contestability risk
> - Calculations (premiums, schedule sections, cash values) must be computed fresh from current test data

### Live Test Output
Each PDF analysis must produce:
1. **Pass/Fail table** — one row per form checked
2. **Flagged issues list** — with page, field, expected rule vs actual value
3. **M&T log entry** — see Section 13

## Section 14 — Do's and Don'ts

### ✅ Do
- Always confirm product, state, form control, channel, and replacement before starting
- Test BOTH sides of every conditional rule (form present AND form absent)
- Verify medical questions individually — never bulk-pass them
- Check carrier name, address, and phone exactly
- Log every FAIL with a Jira ticket number
- Treat field checks as rules — derive values from actual funnel response every time
- Note FL GIWL Perks Rider exclusion — it is always absent in FL GIWL

### ❌ Don't
- Never hardcode expected values for any field
- Never skip the Auth packet generation order check
- Never assume the same form applies across all products — check product-specific rules
- Never mark a FAIL as PASS without a documented fix and re-test
- Never promote to production with an open FAIL and no Jira ticket
- Never test only one side of a conditional rule
- Never confuse GIWL ($0.00 policy fee) with SIWL ($36.00 policy fee)

## Quick Reference — Full Test Checklist

```
PRE-TEST
  [ ] Product confirmed
  [ ] State confirmed
  [ ] Form control confirmed
  [ ] Channel confirmed (DTC / AA)
  [ ] Replacement scenario confirmed (YES / NO)
  [ ] Scenario ID assigned
  [ ] Policy Number recorded in M&T log
  [ ] Tester name logged

PACKET VALIDATION
  [ ] Auth packet — all 8 shared forms present
  [ ] Auth packet — generated at consent page CTA (before app and policy)
  [ ] Application packet — fields match funnel response
  [ ] Application packet — medical questions verified individually
  [ ] Policy packet — correct contract for product + state + form control
  [ ] Riders/benefits — correct state variants, conditional logic applied
  [ ] Buyer's Guide — present, no signature required
  [ ] Carrier HIPAA — correct state variant present

STATE & RULES
  [ ] State-specific form variants applied (Section 11)
  [ ] State endorsements in policy packet where applicable
  [ ] Replacement logic tested both ways (Section 9)
  [ ] Business rules tested both ways (Section 10)
  [ ] CA grace period = 60 days verified if CA test case
  [ ] FL GIWL — Perks Rider confirmed absent
  [ ] FL SIWL — Secondary Addressee section confirmed present

POST-TEST
  [ ] M&T log entry completed
  [ ] All FAILs have Jira ticket numbers
  [ ] CSV export saved
  [ ] Compliance sign-off gates met
```
